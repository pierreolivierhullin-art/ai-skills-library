# Storytelling & Structure Narrative — Minto Pyramid, SCP, Arc Narratif Business

## Overview

Ce document de reference couvre les techniques de storytelling et de structuration narrative pour les presentations professionnelles. La maitrise du recit est le facteur differentiant entre une presentation oubliee et une presentation qui provoque une decision. Un bon storytelling ne signifie pas raconter des anecdotes : c'est l'art de structurer l'information dans un ordre qui cree de la tension, de la comprehension et de la conviction. Utiliser ce guide pour choisir le framework narratif adapte a chaque contexte, construire des ouvertures qui captent l'attention, des transitions fluides et des clotures qui provoquent l'action.

## Minto Pyramid Principle — Structure Top-Down

### Origines et principes

Le Pyramid Principle, developpe par Barbara Minto chez McKinsey dans les annees 1970, est le standard de communication en conseil en strategie, en banque d'investissement et dans les presentations executives. Son principe fondamental : commencer par la conclusion, puis soutenir par des arguments structures.

### Structure detaillee

```
                 ┌─────────────────────────┐
                 │  GOVERNING THOUGHT       │
                 │  (Conclusion / Recomm.)  │
                 └────────┬────────────────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
   ┌──────┴──────┐ ┌──────┴──────┐ ┌──────┴──────┐
   │ Key Line 1  │ │ Key Line 2  │ │ Key Line 3  │
   │ (Argument)  │ │ (Argument)  │ │ (Argument)  │
   └──────┬──────┘ └──────┬──────┘ └──────┬──────┘
          │               │               │
    ┌─────┴─────┐   ┌─────┴─────┐   ┌─────┴─────┐
    │ Support   │   │ Support   │   │ Support   │
    │ Points    │   │ Points    │   │ Points    │
    │ (Preuves) │   │ (Preuves) │   │ (Preuves) │
    └───────────┘   └───────────┘   └───────────┘
```

### Les trois regles fondamentales

1. **Regle de la reponse d'abord (Answer First)** : le premier element presente est toujours la conclusion ou la recommandation. Ne pas construire un raisonnement inductif qui mene a la conclusion : l'audience executive veut la reponse immediatement, puis decide si elle veut entendre les arguments.

2. **Regle MECE (Mutually Exclusive, Collectively Exhaustive)** : les arguments au meme niveau de la pyramide ne se chevauchent pas (mutuellement exclusifs) et couvrent l'ensemble du sujet (collectivement exhaustifs). Si les arguments se chevauchent, la structure n'est pas claire. Si des aspects sont manquants, l'argumentation est incomplete.

3. **Regle de regroupement logique** : les arguments sont regroupes soit par deduction (si A alors B, si B alors C, donc C), soit par induction (A, B et C sont tous des exemples de X). Le regroupement inductif est le plus frequent en presentation : "Trois raisons soutiennent notre recommandation..."

### Application en slides

La Pyramid Principle se traduit en presentation de la maniere suivante :

| Slide | Contenu Pyramid | Exemple |
|---|---|---|
| **Slide 1** | Governing Thought | "Nous recommandons d'investir 5M EUR dans le marche allemand" |
| **Slides 2-4** | Key Line 1 + supports | "Le marche allemand offre le meilleur potentiel de croissance" + donnees marche |
| **Slides 5-7** | Key Line 2 + supports | "Notre produit est adapte aux specificites locales" + benchmark |
| **Slides 8-10** | Key Line 3 + supports | "Le ROI projete est de 3x sur 3 ans" + modele financier |
| **Slide 11** | Recap + Next Steps | Synthese des 3 arguments + plan d'action |

### Test de validation

Pour verifier la solidite de la structure pyramidale, appliquer le test suivant :
- Lire uniquement le governing thought et les key lines (sans les supports). L'histoire est-elle coherente et convaincante ?
- Chaque key line repond-elle au "pourquoi ?" de la governing thought ?
- Chaque support repond-il au "comment le savez-vous ?" de sa key line ?
- Les key lines sont-elles MECE ?

## SCP Framework — Situation, Complication, Proposition

### Principes et origines

Le SCP est le framework narratif le plus utilise en conseil en strategie (McKinsey, BCG, Bain). Il cree un arc narratif naturel en trois actes qui genere de la tension et de l'engagement :

1. **Situation** : etablir le contexte factuel que l'audience connait deja ou accepte facilement. Le but est de creer un terrain d'entente, pas de surprendre.
2. **Complication** : introduire le probleme, le risque, l'opportunite manquee ou le changement qui perturbe la situation. La complication cree la tension narrative — elle rend la suite indispensable.
3. **Proposition** : presenter la solution, la recommandation ou le plan d'action qui resout la complication. La proposition est le coeur de la presentation.

### Structure detaillee

| Acte | Objectif | Questions de redaction | Proportion |
|---|---|---|---|
| **Situation** | Alignement | "Nous savons tous que...", "Le contexte est le suivant..." | 15-20% du deck |
| **Complication** | Tension | "Cependant...", "Le defi est que...", "Ce qui a change..." | 20-25% du deck |
| **Proposition** | Resolution | "Nous proposons de...", "La solution est...", "Notre recommandation..." | 50-60% du deck |

### Variantes du SCP

#### SCQA (Situation-Complication-Question-Answer)

Inserer une question explicite entre la complication et la proposition pour creer un point pivot clair :

```
Situation : "Le marche du cloud croit de 20% par an en Europe"
Complication : "Mais nos parts de marche stagnent depuis 18 mois"
Question : "Comment pouvons-nous capturer notre part de cette croissance ?"
Answer : "En lancant une offre verticale pour le secteur financier"
```

La question cristallise l'enjeu et oriente l'ecoute de l'audience vers la reponse.

#### SCRAP (Situation-Complication-Resolution-Action-Politeness)

Variante pour les emails et les memos courts :
- Situation + Complication en un paragraphe
- Resolution en un paragraphe
- Action en bullet points (qui fait quoi, quand)
- Politeness : formule de cloture et ouverture a la discussion

#### SCR (Situation-Complication-Resolution) pour presentations rapides

Version condensee pour les presentations de 5 minutes :
- 1 slide Situation
- 1-2 slides Complication
- 3-5 slides Resolution
- 1 slide Next Steps

### Erreurs courantes avec le SCP

- **Situation trop longue** : l'audience s'ennuie car elle connait deja le contexte. Maximum 3-4 slides de situation.
- **Complication faible** : si la complication ne genere pas de tension, la proposition n'a pas d'urgence. La complication doit etre ressentie : "Si nous ne faisons rien, voici ce qui va se passer."
- **Proposition non liee a la complication** : la proposition doit resoudre specifiquement le probleme pose dans la complication, pas un autre probleme.
- **Pas de transition claire** : marquer visuellement le passage de la situation a la complication (changement de couleur, slide de transition, mot-cle comme "Cependant" ou "Le defi").

## Hero's Journey Adapte au Business

### Le framework de Joseph Campbell en contexte corporate

Le Hero's Journey est l'arc narratif le plus universel de l'humanite (present dans les mythes, les films, les romans). Adapte au business, il transforme l'audience (ou le client) en heros de l'histoire, et le presentateur (ou l'entreprise) en guide/mentor.

### Les 8 etapes adaptees

| Etape Campbell | Adaptation Business | Contenu de slide |
|---|---|---|
| **Le monde ordinaire** | La situation actuelle du client/marche | Donnees de contexte, tendances, statu quo |
| **L'appel a l'aventure** | L'opportunite ou le defi identifie | Changement de marche, nouveau besoin, probleme emergent |
| **Le refus de l'appel** | Les risques de l'inaction | Ce qui se passe si rien n'est fait (cout du statu quo) |
| **La rencontre du mentor** | L'expertise ou la solution proposee | Presentation de l'approche, de la methodologie, du produit |
| **Le passage du seuil** | La decision de changement | Le moment de decision, le business case |
| **Les epreuves** | Le plan de mise en oeuvre | Etapes, timeline, ressources, risques anticipes |
| **La recompense (l'elixir)** | Les resultats attendus | ROI, benefices, temoignages clients, vision du succes |
| **Le retour** | L'appel a l'action | Prochaines etapes concretes, decision a prendre |

### Quand utiliser le Hero's Journey

- Presentations commerciales et sales decks
- Pitchs investisseurs (la startup est le heros)
- Presentations de transformation interne
- Keynotes et conferences inspirationnelles
- Etudes de cas client

### Quand NE PAS utiliser le Hero's Journey

- Rapports de resultats trimestriels (trop narratif, pas assez factuel)
- Presentations techniques detaillees (le public attend de la precision, pas une histoire)
- Presentations au board ou au COMEX quand le temps est tres limite (preferer Pyramid Principle)

## Techniques d'Ouverture — Capter l'Attention

Les 90 premieres secondes determinent l'engagement de l'audience pour le reste de la presentation. L'ouverture doit sortir l'audience de sa passivite et creer un contrat d'attention.

### 8 techniques d'ouverture eprouvees

1. **La statistique choc** : une donnee surprenante qui remet en question une croyance. "90% des presentations sont oubliees dans les 72 heures. Celle-ci ne le sera pas, et voici pourquoi."

2. **La question rhetorique** : une question qui engage l'audience mentalement. "Combien de decisions a 1M EUR sont prises sur la base d'un deck PowerPoint ? Dans cette salle, probablement toutes."

3. **L'anecdote concrete** : une histoire vraie et courte (30-60 secondes) qui illustre le probleme ou l'opportunite. L'anecdote doit etre specifique, pas generique.

4. **Le contraste avant/apres** : montrer un ecart frappant entre deux etats. "Il y a 18 mois, notre NPS etait a 12. Aujourd'hui, il est a 67. Voici ce qui a change."

5. **La citation d'autorite** : une citation d'un expert reconnu ou d'un client qui pose le sujet. Afficher la citation en plein ecran avec la photo de l'auteur.

6. **L'image plein ecran** : une photo ou une illustration impactante qui cree une reaction emotionnelle. Laisser l'image parler 3-5 secondes avant de commenter.

7. **La promesse explicite** : dire clairement ce que l'audience va gagner. "Dans les 15 prochaines minutes, vous allez comprendre exactement pourquoi nous devons investir dans X, et vous aurez toutes les donnees pour decider."

8. **Le silence** : commencer par 3-5 secondes de silence apres avoir affiche la premiere slide. Le silence cree de la tension et capte l'attention mieux qu'aucun mot.

### Ouvertures a eviter

- "Bonjour, je suis [nom], je suis [titre], et aujourd'hui je vais vous parler de..." — trop predictible, zero tension narrative.
- "Avant de commencer, quelques mots sur notre entreprise..." — l'audience ne s'interesse pas a votre entreprise, elle s'interesse a son probleme.
- "Desole, mes slides ne sont pas tres belles..." — autodenigrement qui detruit la credibilite avant meme de commencer.
- S'excuser pour quoi que ce soit dans les 2 premieres minutes.

## Techniques de Transition

Les transitions entre sections sont critiques pour maintenir le fil narratif et l'attention de l'audience.

### Transitions verbales

| Type | Exemple | Usage |
|---|---|---|
| **Pont logique** | "Maintenant que nous avons vu le probleme, voyons la solution" | Entre complication et proposition |
| **Recapitulation** | "Nous avons donc identifie trois tendances cles. Voyons comment en tirer parti" | Apres une section dense |
| **Question pivot** | "La question est : comment transformer cette menace en opportunite ?" | Pour creer de la tension |
| **Annonce** | "Je vais maintenant vous montrer les donnees qui soutiennent cette affirmation" | Avant une section de preuves |

### Transitions visuelles

- **Slide de section (divider)** : une slide avec le titre de la section suivante, visuellement distincte (couleur de fond differente, typographie plus grande). Creer une respiration de 2-3 secondes sur cette slide.
- **Barre de progression** : un indicateur visuel discret (barre ou points) en bas ou en haut de chaque slide qui montre la progression dans la presentation. Utile pour les presentations longues (> 20 minutes).
- **Recap slide** : avant de changer de section, afficher un slide recapitulant les points cles de la section qui vient d'etre presentee. Technique particulierement efficace apres des sections de donnees denses.

### Transition par callback

Faire reference en fin de presentation a un element de l'ouverture pour creer une boucle narrative satisfaisante. Si l'ouverture contenait une statistique choc, y revenir en conclusion avec la reponse ou la transformation proposee.

## Data Storytelling — Raconter une Histoire avec les Donnees

### Principes du data storytelling en presentation

Le data storytelling n'est pas de la data visualization : c'est l'utilisation des donnees comme preuve dans un recit. La donnee seule ne convainc pas — c'est la donnee contextualisee, annotee et integree dans un arc narratif qui change les decisions.

### Le framework CAA (Contexte-Analyse-Action)

Pour chaque slide de donnees, appliquer ce framework :

1. **Contexte** : d'ou vient cette donnee ? Quelle est la source, la periode, le perimetre ? Formuler dans le titre ou le sous-titre.
2. **Analyse** : que revele cette donnee ? Quel insight peut-on en tirer ? Formuler dans l'annotation directe sur le graphique.
3. **Action** : que faut-il faire avec cet insight ? Formuler dans un call-out ou un encart sur la slide.

### Patterns narratifs pour les donnees

| Pattern | Structure | Usage ideal |
|---|---|---|
| **Tendance** | "De X a Y sur la periode Z" | Croissance, evolution, progression |
| **Comparaison** | "A est superieur a B de X%" | Benchmark, alternatives, concurrence |
| **Composition** | "X est compose de A (30%), B (50%), C (20%)" | Repartition, mix, segmentation |
| **Ecart** | "L'objectif etait X, le realise est Y" | Performance, gap analysis |
| **Correlation** | "Quand X augmente, Y augmente aussi" | Causalite, drivers, leviers |
| **Anomalie** | "Tout est stable sauf Z qui a change de X%" | Alertes, points d'attention, opportunites |

### Regles d'annotation

- **Annoter le point cle directement sur le graphique**, pas dans une legende separee. L'oeil ne doit pas faire l'aller-retour entre le graphique et la legende.
- **Utiliser des fleches, encadres ou cercles** pour pointer l'element important dans un graphique.
- **Chiffre cle en grand** : si la slide veut communiquer un chiffre unique ("+40%", "2.3M EUR"), l'afficher en tres grande taille (48-72pt) au centre de la slide avec le contexte en plus petit en dessous.

## Techniques de Cloture

### Les 6 types de cloture

1. **Call to Action explicite** : la cloture la plus efficace en contexte business. Formuler exactement ce que l'audience doit faire, quand et comment. "Nous avons besoin de votre approbation sur le budget de 500K EUR d'ici vendredi."

2. **Recapitulation des messages cles** : lister les 3 points cles a retenir. Utile pour les presentations informatives ou educatives. Limiter a 3 points maximum (au-dela, l'audience ne retient rien).

3. **Vision du succes** : decrire l'etat futur apres la mise en oeuvre de la recommendation. "Dans 12 mois, si nous executons ce plan, nous serons le numero 2 du marche avec une marge de 25%."

4. **Question ouverte** : poser une question qui invite a la reflexion et a la discussion. "La question n'est pas si nous devons changer, mais a quelle vitesse."

5. **Callback** : revenir a l'anecdote, la statistique ou la question de l'ouverture pour boucler la boucle narrative. Cree un effet de completude satisfaisant.

6. **Citation inspirationnelle** : une citation pertinente qui resume l'esprit de la presentation. A utiliser avec parcimonie et uniquement si la citation est reellement pertinente (pas un cliche motivationnel generique).

### Clotures a eviter

- "Voila, c'est tout, des questions ?" — la presentation se termine en anti-climax. Toujours avoir une derniere slide de cloture avec un message fort avant d'inviter aux questions.
- Ajouter des informations nouvelles dans la cloture — la cloture recapitule et propulse vers l'action, elle n'introduit pas de nouveaux elements.
- Depasser le temps imparti et precipiter la conclusion — mieux vaut couper une section du corps que de bacler la cloture.

## Analyse et Adaptation a l'Audience

### Matrice d'analyse d'audience

Avant chaque presentation, remplir cette matrice pour chaque segment d'audience :

| Critere | Questions a se poser |
|---|---|
| **Qui** | Quels roles, quels niveaux hierarchiques, combien de personnes ? |
| **Connaissance** | Quel est leur niveau de connaissance du sujet ? Expert, intermediaire, novice ? |
| **Attentes** | Qu'esperent-ils apprendre ou obtenir de cette presentation ? |
| **Objections** | Quelles sont leurs objections probables ou leurs preoccupations ? |
| **Decision** | Ont-ils le pouvoir de decider ? Sont-ils des influenceurs ou des decideurs ? |
| **Contexte** | Dans quel contexte organisationnel, politique ou emotionnel se trouvent-ils ? |
| **Action** | Quelle action specifique attendez-vous d'eux apres la presentation ? |

### Adaptation du contenu par type d'audience

| Audience | Style | Contenu | Rythme | Duree ideale |
|---|---|---|---|---|
| **C-Level / Board** | Direct, conclusif | Recommandation d'abord, donnees en backup | Rapide, 1-2 min/slide | 10-15 min + Q&A |
| **Investisseurs** | Ambitieux, chiffre | Vision, traction, TAM/SAM/SOM, team, ask | Structure, 1 min/slide | 15-20 min |
| **Managers / Middle management** | Pragmatique, detaille | Impact sur leur perimetre, plan operationnel | Modere, 2 min/slide | 20-30 min |
| **Equipes techniques** | Precis, factuel | Architecture, specs, trade-offs | Flexible, Q&A integre | 30-45 min |
| **Clients / prospects** | Centre sur eux | Leur probleme, votre solution, preuves | Conversationnel | 15-20 min |
| **Grand public / conference** | Inspirant, visuel | Histoires, insights, visuels impactants | Dynamique, 30s-1min/slide | Selon format |

## Structures de Pitch Deck

### Pitch Deck Investisseurs (Modele Sequoia)

La structure classique recommandee par Sequoia Capital, adaptee et modernisee :

| # | Slide | Contenu |
|---|---|---|
| 1 | **Cover** | Nom, tagline, logo |
| 2 | **Problem** | Le probleme que vous resolvez (douleur client specifique) |
| 3 | **Solution** | Comment vous resolvez le probleme (valeur unique) |
| 4 | **Why Now** | Pourquoi le timing est ideal (tendance, regulation, technologie) |
| 5 | **Market Size** | TAM, SAM, SOM avec methodologie bottom-up |
| 6 | **Product** | Demo, screenshots, fonctionnalites cles |
| 7 | **Traction** | Metriques cles (MRR, croissance, clients, NPS) |
| 8 | **Business Model** | Comment vous gagnez de l'argent (pricing, unit economics) |
| 9 | **Competition** | Positionnement concurrentiel (pas de quadrant magique) |
| 10 | **Team** | Fondateurs, expertise, parcours pertinents |
| 11 | **Financials** | Projections 3-5 ans, hypotheses cles |
| 12 | **Ask** | Montant recherche, utilisation des fonds, timeline |
| 13 | **Vision** | Ou vous serez dans 5 ans |

### Sales Deck (Zuora Structure)

Structure optimisee pour les presentations commerciales :

| # | Slide | Contenu |
|---|---|---|
| 1 | **Le changement** | Un grand shift dans le monde qui affecte le prospect |
| 2 | **Les gagnants et les perdants** | Qui s'adapte et qui stagne |
| 3 | **La terre promise** | L'etat futur ideal pour le prospect |
| 4 | **Les obstacles** | Ce qui empeche le prospect d'atteindre cet etat |
| 5 | **La solution** | Votre approche pour surmonter ces obstacles |
| 6 | **Les preuves** | Cas clients, ROI, temoignages |
| 7 | **Next Steps** | Etapes concretes pour avancer |

### Internal Strategy Deck

Structure pour les presentations de strategie interne :

| # | Section | Contenu | Slides |
|---|---|---|---|
| 1 | **Executive Summary** | Recommandation, chiffres cles, ask | 1-2 |
| 2 | **Context & Market** | Tendances, donnees de marche, benchmarks | 3-5 |
| 3 | **Diagnostic** | Analyse de la situation actuelle, forces/faiblesses | 3-5 |
| 4 | **Strategic Options** | Options evaluees avec criteres | 3-4 |
| 5 | **Recommendation** | Option recommandee avec justification | 2-3 |
| 6 | **Implementation** | Plan d'action, timeline, ressources | 3-5 |
| 7 | **Financial Impact** | Business case, ROI, budget | 2-3 |
| 8 | **Risks & Mitigation** | Risques identifies et plans de mitigation | 1-2 |
| 9 | **Next Steps** | Decisions a prendre, calendar, responsables | 1 |

## Executive Summary Slide — L'Art de la Synthese

### Principes

L'executive summary slide est souvent la slide la plus importante du deck. Elle doit permettre a un decideur presse de comprendre l'essentiel en 30 secondes sans lire le reste de la presentation.

### Structure recommandee

```
┌────────────────────────────────────────────────────┐
│  RECOMMANDATION EN UNE PHRASE (titre affirmatif)    │
│                                                    │
│  ┌─────────────────────────────────────────────┐   │
│  │ Contexte (2 lignes max)                      │   │
│  │ • Fait cle 1                                 │   │
│  │ • Fait cle 2                                 │   │
│  └─────────────────────────────────────────────┘   │
│                                                    │
│  ┌─────────────────────────────────────────────┐   │
│  │ Trois arguments cles                         │   │
│  │ 1. Argument + donnee de support              │   │
│  │ 2. Argument + donnee de support              │   │
│  │ 3. Argument + donnee de support              │   │
│  └─────────────────────────────────────────────┘   │
│                                                    │
│  ┌─────────────────────────────────────────────┐   │
│  │ Ask / Decision attendue                      │   │
│  │ → Action specifique + montant + deadline     │   │
│  └─────────────────────────────────────────────┘   │
└────────────────────────────────────────────────────┘
```

### Regles de l'executive summary

- La recommandation doit etre claire et directe ("Nous recommandons d'investir 5M EUR dans le marche allemand d'ici Q3 2026")
- Chaque argument est soutenu par au moins une donnee chiffree
- Le "ask" specifie exactement ce qui est attendu de l'audience
- L'executive summary ne contient pas de graphiques (seulement du texte structure)
- Elle est autonome : un lecteur qui ne lit que cette slide doit comprendre l'essentiel

## Storytelling Multicanal — Adaptation du Recit

### Presentation live vs deck envoye par email

Un meme contenu necessite deux versions differentes :

| Critere | Deck en live (avec presentateur) | Deck envoye (sans presentateur) |
|---|---|---|
| **Densite de texte** | Minimale (le presentateur commente) | Plus elevee (le texte remplace la voix) |
| **Action titles** | Courts et percutants | Plus detailles et auto-explicatifs |
| **Annotations** | Optionnelles (le presentateur pointe) | Essentielles (pas de main pour guider) |
| **Notes** | Sous les slides (speaker notes) | Integrees dans les slides |
| **Nombre de slides** | 1-2 par minute de presentation | Pas de contrainte de duree |
| **Animations** | Utilisables pour le progressive disclosure | A supprimer (ne fonctionnent pas en PDF) |

### Adaptation pour les formats courts

- **Elevator pitch (60 secondes, 1-3 slides)** : Probleme + Solution + Preuve en 3 phrases
- **Lightning talk (5 minutes, 8-10 slides)** : SCP compresse avec une seule preuve par argument
- **Presentation standard (15-20 minutes, 15-25 slides)** : SCP ou Pyramid complet
- **Workshop (60+ minutes, 30-50 slides)** : Structure modulaire avec exercices et interactions
